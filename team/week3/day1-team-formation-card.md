# Week 3｜Team Formation Card（初稿）

> 本卡片记录 Week 3 Day 1 已确认信息。团队名称、协作工具、同步时间和 Dev 队友补充后再更新，不将未确认事项写成既定事实。

## 团队目标

在 Monad Testnet 完成一个 **Account Abstraction Practice Lab（账户抽象实践与 Proof Demo）**。

Demo 让用户理解 Safe 1.4.1、Safe 4337 Module 与 EntryPoint v0.7 的职责，查看预设账户抽象操作的目的、Owner 签名边界和可验证链上证据。团队不做真实资产操作，也不把自动执行包装成已实现能力。

## 初步用户场景

目标用户是希望理解 Monad 上智能账户、账户抽象或受限账户交互的学习者和开发者。

用户完成的核心动作是：打开 Demo，选择一个预设案例，查看它要完成什么、谁拥有权限、是否需要用户确认、哪些信息来自 Monad Testnet，并跳转区块浏览器核查证据。

## 本周 Scope

### Must Have

- 一个可访问的 Demo 页面或可复现的前端原型；
- 至少一个预设 Monad Testnet 账户交互 / 基础设施验证案例；
- 清晰展示操作说明、权限边界、链上验证链接和 Known Issues；
- README 或 Demo 说明，明确真实部分与 mock 部分。

### Nice to Have

- 更完整的交互引导；
- 3–5 位用户测试后的改进；
- Demo 录屏和更完整的展示页面。

### Not This Week

- 主网或真实资产操作；
- 私钥、助记词或敏感配置的收集与展示；
- 公开 Sponsor、生产级 relayer 或任意 calldata 执行；
- 承诺 Safe Passkeys 已在 Monad Testnet 获得官方模块支持；
- 同时实现多个协议或完整通用 Agent。
