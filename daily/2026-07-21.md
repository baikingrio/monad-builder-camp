### 3. 补齐第一个可演示的技术链路

我在 `experiments/erc4337-frontend` 的 `/safe-4337-monad` 页面增加了一个 **预设零 value UserOperation 学习案例**。它会展示：

- 不使用真实 Safe 地址的预设目标；
- `value = 0` 与空 calldata 的说明；
- Safe 存在、Owner 明确签名、Bundler 模拟、Paymaster 策略或自付 gas 的发送前检查；

页面仍只做公开 bytecode 的读取验证：不连接钱包、不创建 Safe、不请求签名、不发送 UserOperation。预设案例不是交易、不是可执行授权，也没有被写成真实交易 hash 或真实 Safe。

### 5. 尝试将 `/safe-passkeys` 迁移到 Monad Testnet（实验分支）

在 `experiments/erc4337-frontend` 上把 Sepolia Passkeys 教程扩展为 **双网络**：默认 Sepolia，`/safe-passkeys?network=monad` 或页面切换为 **Monad Testnet**。文档见 [SAFE_PASSKEYS_PRACTICE.md](../experiments/erc4337-frontend/SAFE_PASSKEYS_PRACTICE.md)。

**链上与 SDK 边界（调研结论）：**

- Monad 上 **Shared Signer**、**FCL Verifier**、**AddModulesLib**、**EntryPoint v0.6/v0.7** 等有 bytecode；canonical **SafeWebAuthnSignerFactory** 在 `0x1d31…` **无代码**。
- `@safe-global/safe-modules-deployments` 无 `10143`；Relay Kit **v3 仅支持 Safe Modules 0.2.0**，故 Passkeys 实验栈用 **EntryPoint v0.6 + Module v0.2.0**（`0xa581…`），与 `/safe-4337-monad` 只读页引用的 **v0.7 + `0x75cf…`** 文档栈刻意分开。
- Protocol Kit 的 Passkey 路径不消费 Relay 的 `customContracts`；新增 `safePasskeyMonadSdkPatch.ts` 注入 `contractNetworks`，并在 Monad 上 **跳过 factory.getSigner**，统一以 **Shared Signer** 作为 1/1 Owner 推导地址。

**Pimlico / Bundler：**

- Monad 端点统一为 `v2/monad-testnet/rpc?add_balance_override&apikey=…`（`pimlicoEndpoints.ts`）。
- 403 / 401 由真实 Bundler 调用失败时的 `formatSafePasskeyError` 提示 Dashboard 启用 **Monad Testnet** 与 **Allowed origins**（已去掉页面侧主动 `eth_chainId` 探测，避免刷配额）。

**UserOperation 发送调试：**

- 练习目标改为 Monad 上已有 **Check-in 合约**（`checkIn()`），替代 Sepolia 教程 NFT。
- Pimlico `-32500`：`preVerificationGas` 不足（required ~8.4M vs got ~2.5M）；新增 `safePasskeyFeeEstimator`（约 **4× + 下限 9M**）供 Passkey + 首笔 deploy 使用。

**新增 / 主要改动文件：**

- `app/lib/safePasskeyNetworks.ts`、`safePasskeyMonadSdkPatch.ts`、`pimlicoEndpoints.ts`、`safePasskeyFeeEstimator.ts`
- `app/pages/safe-passkeys.vue`（网络切换、文案）
- `safePasskeyConfig.ts`、`safePasskeyActions.ts`、`safe4337MonadConfig.ts`

**验证（迁移后）：**

- `npm test`：**53 / 53** 通过（含 passkeys、Pimlico URL、fee estimator 等）。
- Monad Passkeys **全流程是否能在 Pimlico 赞助下跑通**仍依赖 Dashboard 链权限与 Paymaster policy；本地已越过 Factory 地址、EntryPoint 版本不匹配、Shared Signer 误调 `getSigner`、preVerificationGas 过低等阻塞点，**尚未在对话中确认链上成功 receipt**。
