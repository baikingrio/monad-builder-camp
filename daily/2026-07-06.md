## 今天印象最深的几个点

### 1. Monad 测试币领水体验很友好

今天第一个很明显的感受是：Monad Testnet 的测试币领取非常方便。

相比有些公链测试网水龙头需要复杂验证、绑定账号、排队，甚至经常额度不足，Monad 的领水过程对新手更友好。对刚进入链上开发的人来说，测试币门槛越低，越容易马上进入“动手实践”的状态。

这点很重要，因为 Week 1 的核心目标不是研究复杂协议，而是让大家尽快完成第一笔链上操作。如果一开始就卡在水龙头上，会很影响学习体验。

### 2. 合约源码验证不需要 API Key，很顺滑

第二个让我印象很深的是源码验证。

过去在一些 EVM 链上做 `forge verify-contract`，经常需要先准备 Etherscan 类 API Key，还要确认 verifier URL、chain config、constructor args 等。如果只是学习和测试，这一步很容易让新手卡住。

这次在 MonadVision 上验证合约源码，可以直接通过 Sourcify 路径完成：

```shell
forge verify-contract \
  0xef694e5411A70d05270722A38Ea9Ef242E3afcf2 \
  src/CampToken.sol:CampToken \
  --chain 10143 \
  --verifier sourcify \
  --verifier-url https://sourcify-api-monad.blockvision.org/ \
  --constructor-args $(cast abi-encode "constructor(string,string,uint256)" "Camp Token" "CAMP" 1000000000000000000000000) \
  --watch
```

成功后显示：

```text
Status: exact_match
```

这个体验很好：不需要额外申请 API Key，也能让区块浏览器上看到 Verified 源码。对学习者来说，这让“开源合约”和“验证合约”变成了一个自然步骤，而不是额外负担。

### 3. README 记录比单纯部署更重要

今天我也再次感受到：链上实验如果只做一次部署，很容易过几天就忘了细节。

所以我把关键内容都写进了 `experiments/monad-playground/README.md`：

- Monad Testnet Chain ID 和 RPC；
- 水龙头使用方式；
- `forge script` 部署命令；
- `cast call` 查询命令；
- `forge verify-contract` 源码验证命令；
- 已部署合约地址、交易 hash 和 Explorer 链接。

这样后续不管是做 Week 1 任务提交、Mini Demo 0，还是 Week 4 Hackathon 项目，都可以直接引用这份记录。
