## 今天学习 / 完成了什么

今天主要阅读了 Safe SDK 官方 Overview，并单独整理了一份 Safe SDK 学习笔记。

我先厘清了 Safe 和普通 EOA 钱包的区别。EOA 通常由一把私钥直接控制；Safe 是链上的智能合约账户，可以把 owner、threshold、多签确认、Module、Guard 等规则写进账户本身。

Safe SDK 将能力分成四个 Kit：

- Starter Kit：TypeScript 接入 Safe 的默认入口，适合快速处理创建账户和常见交易流程；
- Protocol Kit：直接操作 Safe 合约能力，例如部署 Safe、调整 owner / threshold、构造和执行交易；
- API Kit：连接 Safe Transaction Service，用于交易提案、签名收集、待确认交易和历史查询；
- Relay Kit：面向 ERC-4337、UserOperation 和 relay 等账户抽象执行路径。

我也梳理了一笔典型 Safe 多签交易的流程：

```text
构造交易 → 创建 Safe Transaction → 提案与共享
→ Owner 分别确认 → 达到 threshold → Safe 合约执行
```

## 关键理解

今天最重要的收获是：Safe SDK 不只是“调用一个钱包”的工具，而是把智能账户、链上授权规则和多人交易协作拆成了不同层次。

后续如果做 Safe 应用，应该先从 Starter Kit 入门；需要直接控制 Safe 合约时再看 Protocol Kit；需要多签成员协作与交易队列时使用 API Kit；需要 ERC-4337 / UserOperation 路径时再研究 Relay Kit。

另外，Gas 代付或 relay 只是在交易提交和 Gas 支付层改善体验，并不能绕过 Safe 的 owner、threshold、Module 或 Guard 等授权规则。
